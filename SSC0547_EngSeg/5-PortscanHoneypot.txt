Lab. 5 (parte 1) - Portscan

================================================
1. Explique o funcionamento dos comandos de redirecionamento de portas.

Os comandos de redirecionamento de porta funcionam recebendo como parâmetros duas portas e dois computadores - quando um dispositivo tentar acessar a primeira porta do primeiro computador, será feito o redirecionamento para a segunda porta do segundo computador.

================================================
2. Pesquise comandos interessantes para se utilizar com o iptables e outras opções de uso da ferramenta nmap.

Comandos para se utilizar com iptables: pode-se bloquear algum IP específico, com iptables ' -A INPUT -s xxx.xxx.xxx.xxx -j DROP '. Também é possível bloquear algum site específico a partir de seu CIDR, com 'iptables -A OUTPUT -p tcp -d xxx.xxx.xxx.xxx/xx -j DROP'. O nmap pode ser utilizado para várias fins, como identificar hosts em uma rede, listando os hosts que respondem a alguns requests TCP, ou que possuem alguma porta em particular aberta. Também pode ser utilizado para determinar a detecção de SO.

================================================
3. Pesquise outras formas de levantar informações sobre o computador de uma possível vítima.

Também pode-se levantar informações sobre o computador de uma possível vítima utilizando backdoor ou bots, além de outros softwares similares ao nmap, como Advanced Port Scanner, Advanced IP Scanner, Angry IP Scanner, Dipiscan, dentre outros.

================================================
4. Pesquise possíveis configurações de firewall para bloquear port-scans. Porque elas não são muito efetivas? Quais as melhores formas de realizar essa proteção?

É possível configurar o firewall para bloquear portscans por meio de filtros de firewall, porém, isto pode não ser totalmente eficiente, visto que as portas bloqueadas violam as regras de conduta do TCP/IP, o que pode deixar algumas portas não bloqueadas. Também pode-se configurar o firewall para redirecionar portas abertas para honeypots ou hosts vazios. Esta proteção também pode ser realizada utilizando-se o nmap para escanear as portas do próprio dispositivo, ou então utilizando-se a ferramenta PSAD.

================================================
5. Pesquise sobre scanners de rede. Utilize o laboratório 4 com o zebra ativo e, se possível, tente usar o scanner neste laboratório.

"Scanners de rede", como o nome diz, são ferramentas utilizadas para escanear redes, tanto corporativas de larga escala quanto redes domésticas. Estes mostrarão todos os recursos compartilhados, como recursos WEB, FTP, dentre outros.

Lab. 5 (parte 2) - Honeypot


================================================
1. Altere o arquivo de configuração mel.conf para aceitar uma nova máquina. Use a personalidade “Linux 2.4.7 (X86)”.

Alteração feita do arquivo "/etc/honeypot/mel.conf".

================================================
2. Quais as vantagens de usar o honeypot baseado em aplicação? Qual desvantagem foi percebida?

A principal vantagem de se utilizar honeypot baseado em aplicação é o custo, pois não são necessários dispositivos para simular redes. A principal desvantagem é que, por não ser uma rede real, a "armadilha" pode ser detectada pelo atacante.

================================================
3. Proponha uma solução para a criação de uma honeynet.

A criação da honeynet pode ser feita de maneira real (física) ou virtual. Na maneira real são necessários diversos computadores, contendo honeypots, IDS, IPS, firewall dentre outros. Na virtual, todos os componentes de um honeypot podem ser implementados em um mesmo dispositivo - para isto, é utilizado um computador com SO instalado, que dará suporte às máquinas virtuais.

================================================
4. Porque o uso de honeypots ou honeynets é interessante? E as saídas produzidas pelo honeyd (ou seus logs)?

Quando se utiliza honeypots e honeynets, o possível atacante que acredita que está em uma rede ou aplicação, na verdade, está em um ambiente simulado. Com isso, pode ser desgastado tempo deste atacante, que poderia estar sendo usado para invadir redes reais. As saídas produzidas nos logs são interessantes pois permitem que o admin identifique o atacante e faça as devidas precauções de um possível ataque real.
